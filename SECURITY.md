# Security Policy

## Supported versions

`0.1.x` is the current supported line.

## Reporting

Do not report security issues in public GitHub issues.

Use GitHub private vulnerability reporting once it is enabled for the repository. If private reporting is not enabled yet, contact the maintainers directly through GitHub and keep the report non-public until remediation is available.

Include:

- affected version
- operating system and Bun version
- Zig version if the issue touches native build or FFI behavior
- reproduction steps or proof of concept
- impact and suggested mitigation if known

## Response target

Initial acknowledgement target: 3 business days.
